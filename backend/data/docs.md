# 🛡️ GUI-Anything 深度分析手册

> **提示**：本工具主要用于辅助 Android 逆向工程与源码审计。你可以通过「生成 UI Map」把仓库源码抽取为交互图谱，再把图谱（可选）附加到对话中进行定位与修复建议生成。

---

## 1. 核心架构说明
GUI-Anything 会为每个项目（`app_id`）维护一份 UI Map（JSON），并在前端以交互图谱的形式展示。UI Map 的数据来源可以是：

- **真实引擎生成**：从已下载的仓库源码中解析 Activity/Fragment/Dialog、元素与跳转关系
- **Mock 生成（兜底）**：当真实引擎不可用时，系统会自动回退，保证 UI 可用

### 节点类型定义
- **Launcher Activity** (蓝色): 应用的物理入口。
- **Standard Activity** (灰色): 常规业务承载页面。
- **Exported Activity** (红色警告): 具有外部调用风险的组件。

---

## 2. 如何生成 UI Map？

### 2.1 在「交互图谱」抽屉中生成
- 打开侧边栏的 **交互图谱**
- 点击左上角 **生成 UI Map**
- 生成完成后会自动刷新画布

### 2.2 在「后台管理系统」项目详情中生成
- 进入 **管理控制台 → 项目资源管理 → 选择项目**
- 在 **UI Map 状态** 卡片中查看：
  - 是否已生成
  - 更新时间
- 未生成时点击 **生成图谱**；已生成可点 **重新生成**

> 说明：生成任务是异步的，前端会轮询进度；结果会写入 `backend/data/{app_id}.json`，供 UI Map 展示与后续使用。

## 3. 如何进行深度分析（对话区）？
你可以利用模型的推理能力，结合 Issue 与（可选）UI Map/源码检索进行分析与修复建议生成。

### 3.1 添加 ISSUE（推荐）
- 点击输入框下方 **添加 ISSUE**
- 选择要附加的 Issue（将以附件形式并入本次提问上下文）

### 🔍 场景 A：权限追踪
**提问示例**：
`"分析该应用在进入 Simple-Dialer 拨号页面前，是否进行了运行时权限检查？"`

### 🔗 场景 B：逻辑盲区扫描
**提问示例**：
`"图中显示 MainActivity 可以直接跳转到 PayActivity，这是否绕过了身份验证逻辑？"`

---

## 4. 添加图谱（可选）
当你需要模型显式引用 UI Map（例如让它沿跳转链路解释根因、定位到具体页面/交互点）：

- 点击输入框下方 **添加图谱**
- 系统会把当前项目的 UI Map 以附件形式加入本次提问

> 注意：图谱内容可能很大，前端会自动截断；如信息不足可通过「继续」按钮让系统再检索更多源码信息。

---

## 5. 工具调用上限：继续 / 强制输出
当模型需要多轮检索源码（例如多次 `repo_read_file`）时，系统存在工具调用轮次上限。

到达上限后，对话框下方会出现两个按钮：
- **继续**：再增加一段工具轮次，继续检索
- **强制输出**：不再调用工具，基于当前已收集信息直接输出结论与修复建议

---

## 6. 常见问题 (FAQ)
| 问题 | 解决方法 |
| :--- | :--- |
| UI Map 显示为空 / 未生成 | 在「交互图谱」中点击生成，或在「管理控制台 → UI Map 状态」中生成。 |
| 图谱生成失败或回退 mock | 请确认仓库已下载就绪；真实引擎依赖（如 LLM key、解析依赖）缺失会触发回退。 |
| AI 一直在读文件/搜索，无法收敛 | 使用「强制输出」让它基于当前信息给出修复建议；或点「继续」补充更多证据。 |
| 登录失效 | 为了安全，Session 会在 24 小时后过期。 |
